# API Reference Overview

Welcome to the Proxed.AI API reference documentation. These pages provide detailed information about each available API endpoint, including how to authenticate, structure your requests, and interpret responses.

All API endpoints are versioned under `/v1/`.

## Authentication

Most API endpoints require authentication. Public endpoints that **do not** require auth:

*   `GET /health`
*   `GET /v1/openai/models`
*   `GET /v1/openai/{projectId}/models`
*   `GET /v1/anthropic/models`
*   `GET /v1/anthropic/{projectId}/models`
*   `GET /v1/google/models`
*   `GET /v1/google/{projectId}/models`

All other API routes require authentication. See the [API Authentication guide](/authentication) for detailed information on how to authenticate your requests.

## Error Handling

Consistent error responses are provided across the API. See the [API Errors guide](/api-errors) for a comprehensive list of error codes and how to handle them.

## Available Endpoints

Select an endpoint from the navigation sidebar to view its specific documentation, including:

*   **Structured Vision Analysis:** `/v1/vision`
*   **Structured Text Analysis:** `/v1/text`
*   **Structured PDF Analysis:** `/v1/pdf`
*   **Image Generation (Experimental):** `/v1/image`
*   **OpenAI Proxy:** `/v1/openai/{projectId}/*`
*   **Anthropic Proxy:** `/v1/anthropic/{projectId}/*`
*   **Google AI Proxy:** `/v1/google/{projectId}/*`
*   **Public Model Lists:** `/v1/{provider}/models` and `/v1/{provider}/{projectId}/models`
